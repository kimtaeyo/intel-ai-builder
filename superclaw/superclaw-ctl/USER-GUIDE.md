# Inference Edge Server User Guide

This guide shows how to set up the SuperClaw inference edge server on the model-serving workstation.

Run these commands on the workstation, not on the AI PC. The workstation runs the local vLLM model-serving stack, and the SuperClaw Client App connects to it by using two values:

- **Server URL** from `superclaw-ctl up`
- **Connection key** from the full `vllm_api_key` generated by `superclaw-ctl init`

---

## Before You Start

Make sure the model-serving workstation has:

| Requirement | Notes |
|-------------|-------|
| Ubuntu Linux x86_64 | Must run on the server host |
| Docker Engine with Compose plugin | `docker compose` must work |
| Intel GPU | Tested on Intel® Arc B70 Pro |
| Internet access | Required on first setup to pull images and download models |

If the server uses a proxy, configure `HTTP_PROXY`, `HTTPS_PROXY`, and `NO_PROXY` before setup. Include `localhost` and `127.0.0.1` in `NO_PROXY`. If Docker also needs the proxy to pull images, configure Docker proxy settings by following the [Docker proxy documentation](https://docs.docker.com/engine/daemon/proxy/).

---

## Install `superclaw-ctl`

Download the [superclaw-ctl](./binary_build/superclaw-ctl-v1.0.0-linux-x86-64.tar.gz) binary on the model-serving workstation, then install it:

```bash
tar -xzf superclaw-ctl.tar.gz
chmod +x superclaw-ctl
sudo mv superclaw-ctl /usr/local/bin/
```

Verify the command works:

```bash
superclaw-ctl --help
superclaw-ctl version
```

## First-Time Setup

### 1. Initialize the server

Run `init` once. It checks prerequisites, creates config files, generates API keys, and downloads the default models.

```bash
superclaw-ctl init
```

During initialization, `superclaw-ctl` shows the full API key one time:

```text
Generating API keys...
Keys generated. They will be shown once below:

  Key            Value
 ------------------------------------------------------------
  vllm_api_key   <your-api-key>

Config saved to /root/.config/superclaw-ctl
```

Copy and save the full `vllm_api_key`. This is the **connection key** for the SuperClaw Client App.

If you miss it, you can reveal it again later:

```bash
superclaw-ctl keys show --reveal
```

### 2. Start the server

Run `up` to start the model-serving stack:

```bash
superclaw-ctl up
```

The first start can take several minutes while the models load. When startup succeeds, `superclaw-ctl` shows a connection summary:

```text
Connection
URL: http://192.168.1.1:8080
vLLM Model Router: 192.168.1.1:8080
Token: <api-key>...
```

Copy the **URL**. The **Token** shown by `up` is shortened, so do not use it as the Client App connection key. Use the full `vllm_api_key` from `init` or `superclaw-ctl keys show --reveal`.

### 3. Connect the SuperClaw Client App

On the AI PC, open the SuperClaw Client App and enter:

- **URL**: the URL from `superclaw-ctl up`
- **Connection key**: the full `vllm_api_key`

The Client App verifies the connection and registers the inference edge server as your local model source.

### 4. Check server health

```bash
superclaw-ctl status
```

## Common Commands

| Task | Command |
|------|---------|
| First-time setup | `superclaw-ctl init` |
| Start the server | `superclaw-ctl up` |
| Start on another router port | `superclaw-ctl up --router-port 9090` |
| Check health | `superclaw-ctl status` |
| View logs | `superclaw-ctl logs` |
| Follow live logs | `superclaw-ctl logs -f` |
| Stop the server | `superclaw-ctl down` |
| Show the full connection key | `superclaw-ctl keys show --reveal` |
| Run diagnostics | `superclaw-ctl doctor` |

## Troubleshooting

| Symptom | What to do |
|---------|------------|
| `Config file not found` | Run `superclaw-ctl init` first. |
| Forgot the connection key | Run `superclaw-ctl keys show --reveal`. |
| `up` waits at `Waiting for model router to become healthy...` | Usually normal on first start. Wait several minutes while models load. |
| Backends are not healthy after startup | Run `superclaw-ctl up --timeout 1800` to allow more time. |
| Health probes fail with proxy enabled | Add `localhost` and `127.0.0.1` to `NO_PROXY`. |
| Model files are missing or incomplete | Run `superclaw-ctl models download` or `superclaw-ctl models download --verify`. |

### Startup takes several minutes

On first start, vLLM loads the embedding model first and then the chat model. The chat model can take the longest, especially with a large context window.

To watch progress, run this in another terminal:

```bash
superclaw-ctl logs -f
```

Look for these milestones:

| Log line | Meaning |
|----------|---------|
| `[vllm] embed ready` | Embedding model is ready |
| `[vllm] starting chat ...` | Chat model is loading |
| `[vllm] chat ready` | Chat model is ready |
| `registering chat backend` / `registering embed backend` | Backends are being connected to the router |

If logs show an out-of-memory or GPU error and the container keeps restarting, check `superclaw-ctl status`, `docker ps`, and model integrity with:

```bash
superclaw-ctl models download --verify
```

## Advanced Operations

### Models

```bash
superclaw-ctl models list
superclaw-ctl models info Qwen3-Coder-Next
superclaw-ctl models download
superclaw-ctl models download --model qwen3-coder-next
superclaw-ctl models download --verify
```

Models are stored in `~/.models` by default, unless configured otherwise.

### API keys

```bash
superclaw-ctl keys show           # show redacted keys
superclaw-ctl keys show --reveal  # show full key values
superclaw-ctl keys rotate         # generate new keys
```

After rotating keys, restart the server so the new key takes effect:

```bash
superclaw-ctl down
superclaw-ctl up
```

### Configuration

```bash
superclaw-ctl config show
superclaw-ctl config set paths.models_dir /data/models
superclaw-ctl config set images.vllm intel/llm-scaler-vllm:0.15.0
```

Config is stored in `~/.config/superclaw-ctl/config.toml`.

### Cleanup

Preview cleanup before deleting anything:

```bash
superclaw-ctl clean containers --dry-run
superclaw-ctl clean images --dry-run
superclaw-ctl clean config --dry-run
superclaw-ctl clean all --dry-run
```

Run cleanup:

```bash
superclaw-ctl clean containers
superclaw-ctl clean images
superclaw-ctl clean config
superclaw-ctl clean all
```

Note: The models directory is not deleted by cleanup commands.
